# Agentic Plan Caching (APC) — 第一版架构实现设计

> 基于论文 *Agentic Plan Caching: Test-Time Memory for Fast and Cost-Efficient LLM Agents*（NeurIPS 2025）  
> 技术栈：**Redis · LangGraph · FastAPI · PostgreSQL · Prometheus/Grafana**

---

## 1. 设计目标

| 目标 | 指标（论文基准） |
|------|----------------|
| 降低 API 成本 | ↓ 50.31% |
| 降低端到端延迟 | ↓ 27.28% |
| 保持任务准确率 | ≥ 96.61% 最优基线 |
| 缓存机制开销 | ≤ 1.04% 总成本 |

**核心思路**：不缓存"问答对"（Semantic Caching 做法），而是缓存经过抽象的 **计划模板（Plan Template）**。命中时用轻量小模型做上下文适配，未命中时走大模型并在执行后自动生成模板写回缓存。

## 3. LangGraph 图设计

### 3.1 状态定义（AgentState）

```python
from typing import TypedDict, Optional, List, Any
from langgraph.graph import StateGraph, END

class AgentState(TypedDict):
    # 输入
    query: str
    context: Any                    # 外部数据、文档、GUI 环境

    # 关键词 & 缓存
    keyword: Optional[str]
    cache_hit: bool
    plan_template: Optional[dict]   # 命中时获取的模板

    # 执行状态
    current_plan: Optional[str]
    actor_responses: List[str]
    execution_log: List[dict]       # cache miss 时记录完整日志
    iteration_count: int

    # 输出
    final_output: Optional[str]
    is_complete: bool
```

### 3.2 节点实现

#### keyword_node（关键词提取）

```python
async def keyword_node(state: AgentState) -> AgentState:
    """
    使用轻量模型提取任务高层意图关键词。
    "What is FY2019 working capital ratio for Costco?"
        → "working capital ratio"
    成本：约占总成本 0.12%（用 GPT-4o-mini）
    """
    prompt = KEYWORD_EXTRACTION_PROMPT.format(query=state["query"])
    keyword = await lightweight_llm.ainvoke(prompt)  # GPT-4o-mini
    return {**state, "keyword": keyword.strip().lower()}
```

#### cache_lookup_node（Redis 精确查找）

```python
async def cache_lookup_node(state: AgentState) -> AgentState:
    """
    O(1) Redis HGET 查询，精确匹配关键词。
    论文：精确匹配 vs 模糊匹配 —— 精确匹配在 10^6 条目下仍 < 60μs，
    模糊匹配（SentenceTransformer）在 10^5 条目下已达 10s 级别。
    """
    keyword = state["keyword"]
    template_json = await redis_client.hget("apc:plans", keyword)

    if template_json:
        # 命中：更新统计、刷新 LRU 时间戳
        await redis_client.hincrby("apc:stats", "hits", 1)
        await redis_client.zadd("apc:lru", {keyword: time.time()})
        return {**state, "cache_hit": True,
                "plan_template": json.loads(template_json)}
    else:
        await redis_client.hincrby("apc:stats", "misses", 1)
        return {**state, "cache_hit": False}
```

#### cache_hit 路由

```python
def route_after_lookup(state: AgentState) -> str:
    return "small_planner" if state["cache_hit"] else "large_planner"
```

#### small_planner_node（命中时：小模型适配模板）

```python
async def small_planner_node(state: AgentState) -> AgentState:
    """
    Cache Hit 路径：用 LLaMa-3.1-8B 将泛化模板适配为当前任务的具体计划。
    避免调用昂贵的大模型，是降低成本的核心。
    """
    prompt = CACHE_ADAPTATION_PROMPT.format(
        cached_task=state["plan_template"]["task"],
        next_step=state["plan_template"]["steps"][state["iteration_count"]],
        current_task=state["query"],
        past_messages=state["current_plan"],
        past_responses="\n".join(state["actor_responses"]),
    )
    adapted_plan = await small_llm.ainvoke(prompt)  # LLaMa-3.1-8B（本地）
    is_done = adapted_plan.get("answer") is not None

    return {
        **state,
        "current_plan": adapted_plan.get("message"),
        "final_output": adapted_plan.get("answer"),
        "is_complete": is_done,
        "iteration_count": state["iteration_count"] + 1,
    }
```

#### large_planner_node（未命中时：大模型生成计划）

```python
async def large_planner_node(state: AgentState) -> AgentState:
    """
    Cache Miss 路径：调用 GPT-4o 生成完整计划。
    同时将 (plan, context, response) 追加到 execution_log，
    供后续模板生成使用。
    """
    history = build_conversation_history(state)
    plan_response = await large_llm.ainvoke(history)  # GPT-4o

    log_entry = {
        "plan": plan_response.get("message"),
        "context_hash": hash_context(state["context"]),
        "iteration": state["iteration_count"],
    }
    is_done = plan_response.get("answer") is not None

    return {
        **state,
        "current_plan": plan_response.get("message"),
        "final_output": plan_response.get("answer"),
        "is_complete": is_done,
        "execution_log": state["execution_log"] + [log_entry],
        "iteration_count": state["iteration_count"] + 1,
    }
```

#### actor_node（执行计划）

```python
async def actor_node(state: AgentState) -> AgentState:
    """
    Actor LM 接收计划 + 上下文，输出响应。
    本地运行 LLaMa-3.1-8B，无 API 成本。
    """
    response = await actor_llm.ainvoke(
        plan=state["current_plan"],
        context=state["context"],
    )
    updated_log = state["execution_log"].copy()
    if updated_log:
        updated_log[-1]["response"] = response

    return {
        **state,
        "actor_responses": state["actor_responses"] + [response],
        "execution_log": updated_log,
    }
```

#### template_gen_node（写入缓存）

```python
async def template_gen_node(state: AgentState) -> AgentState:
    """
    Cache Miss 完成后异步生成模板并写入 Redis。
    两步过滤：
      1. rule_filter：去除冗余推理步骤，提取 plan/response 核心结构
      2. llm_filter：用 GPT-4o-mini 泛化，去掉公司名、年份等特定信息
    开销：仅占总成本 ~0.88%
    """
    # Step 1: rule-based filter
    raw_template = rule_filter(state["execution_log"])

    # Step 2: LLM-based generalization
    prompt = CACHE_GENERATION_PROMPT.format(trace=json.dumps(raw_template))
    generalized = await lightweight_llm.ainvoke(prompt)
    template = json.loads(generalized)

    # 写入 Redis（HSET + LRU 时间戳）
    keyword = state["keyword"]
    await redis_client.hset("apc:plans", keyword, json.dumps(template))
    await redis_client.zadd("apc:lru", {keyword: time.time()})

    # LRU 容量管理
    await enforce_lru_eviction(max_size=CACHE_MAX_SIZE)

    # 异步持久化到 PostgreSQL
    asyncio.create_task(persist_to_db(keyword, template, state))

    return state
```

### 3.3 图组装

```python
def build_apc_graph() -> StateGraph:
    graph = StateGraph(AgentState)

    # 注册节点
    graph.add_node("keyword",       keyword_node)
    graph.add_node("cache_lookup",  cache_lookup_node)
    graph.add_node("small_planner", small_planner_node)
    graph.add_node("large_planner", large_planner_node)
    graph.add_node("actor",         actor_node)
    graph.add_node("template_gen",  template_gen_node)

    # 入口
    graph.set_entry_point("keyword")

    # 固定边
    graph.add_edge("keyword",       "cache_lookup")

    # 条件边：命中 vs 未命中
    graph.add_conditional_edges(
        "cache_lookup",
        route_after_lookup,
        {"small_planner": "small_planner", "large_planner": "large_planner"},
    )

    # 执行后路由：完成 vs 继续迭代
    graph.add_conditional_edges(
        "small_planner",
        lambda s: END if s["is_complete"] else "actor",
    )
    graph.add_conditional_edges(
        "large_planner",
        lambda s: END if s["is_complete"] else "actor",
    )
    graph.add_conditional_edges(
        "actor",
        route_after_actor,  # 完成 → template_gen（miss）或 END（hit）
    )
    graph.add_edge("template_gen", END)

    # 持久化 Checkpointer（支持中断恢复）
    checkpointer = RedisCheckpointer(redis_client)
    return graph.compile(checkpointer=checkpointer)


def route_after_actor(state: AgentState) -> str:
    if state["is_complete"]:
        return "template_gen" if not state["cache_hit"] else END
    return "small_planner" if state["cache_hit"] else "large_planner"
```

## 4. Redis 数据结构设计

```
# ── 计划缓存（核心）──────────────────────────────────
HSET apc:plans  <keyword>  <JSON plan template>
  └─ 字段即关键词，值为 JSON 序列化的 plan template

# ── LRU 排序集（容量管理）───────────────────────────
ZADD apc:lru    <timestamp>  <keyword>
  └─ score = 最近访问时间戳，ZRANGEBYSCORE 驱逐最旧条目

# ── 命中率统计 ───────────────────────────────────────
HINCRBY apc:stats  hits    1
HINCRBY apc:stats  misses  1
HINCRBY apc:stats  total   1

# ── 每关键词细粒度统计 ───────────────────────────────
HINCRBY apc:keyword_stats:<keyword>  hit_count  1
HSET    apc:keyword_stats:<keyword>  last_used  <timestamp>
HSET    apc:keyword_stats:<keyword>  cost_saved <float USD>

# ── 持久化 ───────────────────────────────────────────
# redis.conf
appendonly yes          # AOF 持久化（防止重启丢失）
appendfsync everysec    # 每秒 fsync（性能与安全平衡）
save 900 1              # RDB 兜底快照
```

### LRU 驱逐实现

```python
async def enforce_lru_eviction(max_size: int = 100):
    """
    容量超限时，驱逐最旧的 (max_current - max_size) 个条目。
    论文：cache size=100 时命中率约 46%，继续增大收益递减。
    """
    current_size = await redis_client.hlen("apc:plans")
    if current_size <= max_size:
        return

    evict_count = current_size - max_size
    # 获取最旧的 evict_count 个关键词
    oldest = await redis_client.zrange("apc:lru", 0, evict_count - 1)
    if oldest:
        await redis_client.hdel("apc:plans", *oldest)
        await redis_client.zrem("apc:lru", *oldest)
```

## 5. FastAPI 入口层

```python
from fastapi import FastAPI, WebSocket
from contextlib import asynccontextmanager

app = FastAPI(title="APC Agent Service")

@asynccontextmanager
async def lifespan(app):
    app.state.graph = build_apc_graph()
    app.state.redis = await create_redis_pool()
    yield
    await app.state.redis.close()

@app.post("/agent/run")
async def run_agent(request: AgentRequest):
    """标准同步调用，返回完整输出。"""
    initial_state = AgentState(
        query=request.query,
        context=request.context,
        keyword=None, cache_hit=False, plan_template=None,
        current_plan=None, actor_responses=[], execution_log=[],
        iteration_count=0, final_output=None, is_complete=False,
    )
    result = await app.state.graph.ainvoke(
        initial_state,
        config={"configurable": {"thread_id": request.session_id}},
    )
    return {"output": result["final_output"], "cache_hit": result["cache_hit"]}


@app.websocket("/agent/stream")
async def stream_agent(ws: WebSocket):
    """流式输出，每个 Actor 响应实时推送。"""
    await ws.accept()
    data = await ws.receive_json()
    initial_state = build_initial_state(data)

    async for event in app.state.graph.astream_events(initial_state, version="v2"):
        if event["event"] == "on_chain_stream":
            chunk = event["data"].get("chunk", {})
            if actor_response := chunk.get("actor_responses"):
                await ws.send_json({"type": "token", "data": actor_response[-1]})
        elif event["event"] == "on_chain_end":
            await ws.send_json({"type": "done", "cache_hit": event["data"]["cache_hit"]})
            break


@app.get("/cache/stats")
async def get_cache_stats():
    stats = await redis_client.hgetall("apc:stats")
    total = int(stats.get("total", 1))
    hits  = int(stats.get("hits",  0))
    return {
        "hit_rate":    hits / total,
        "total_queries": total,
        "cache_size":  await redis_client.hlen("apc:plans"),
    }


@app.post("/cache/warmup")
async def warmup_cache(samples: List[WarmupSample]):
    """
    冷启动缓解：预加载离线样本到缓存。
    论文：pre-warming 可显著缓解冷启动成本。
    """
    for sample in samples:
        template = await generate_template_from_sample(sample)
        await redis_client.hset("apc:plans", sample.keyword, json.dumps(template))
    return {"warmed": len(samples)}
```

## 6. 模板生成——规则过滤器

```python
def rule_filter(execution_log: List[dict]) -> dict:
    """
    从执行日志提取结构化模板，去除冗余推理文本。
    保留：plan 指令、expected response 类型、最终 answer 结构。
    丢弃：verbose CoT 推理、具体数值、实体名称（由后续 LLM filter 处理）。
    """
    steps = []
    for i, entry in enumerate(execution_log):
        plan = entry.get("plan", "")
        response = entry.get("response", "")

        # 提取句子级别的指令（去除解释性文字）
        plan_sentences = extract_action_sentences(plan)
        response_type  = infer_response_type(response)   # e.g. "numeric", "list"

        steps.append({
            "step": i + 1,
            "plan_directive": " ".join(plan_sentences),
            "expected_response_type": response_type,
        })

    return {
        "task": "{{task_description}}",   # 占位符，供 LLM filter 填充
        "steps": steps,
        "total_iterations": len(execution_log),
    }
```

## 7. 自适应降级策略（额外优化）

论文建议：当命中率持续为零时，动态关闭缓存以节省 keyword extraction 开销。

```python
class AdaptiveCacheController:
    """
    监控近 N 次请求的命中率，动态决定是否启用缓存。
    当命中率 < threshold 超过 window 次请求时，暂停缓存（跳过 keyword_node 和 cache_lookup_node）。
    """
    def __init__(self, window: int = 50, threshold: float = 0.05):
        self.window    = window
        self.threshold = threshold
        self.recent: deque = deque(maxlen=window)

    async def should_use_cache(self) -> bool:
        stats = await redis_client.hgetall("apc:stats")
        # 仅用最近 window 次数据
        recent_rate = sum(self.recent) / len(self.recent) if self.recent else 1.0
        return recent_rate >= self.threshold

    def record(self, hit: bool):
        self.recent.append(1 if hit else 0)
```

## 10. 设计取舍与后续方向

| 决策 | 选择 | 理由 |
|------|------|------|
| 缓存匹配 | **精确关键词匹配** | 模糊匹配引入假阳性，且 10^5 级别延迟达秒级 |
| 关键词提取模型 | **GPT-4o-mini** | 成本仅 0.12%，效果与 GPT-4o 相当 |
| 模板适配模型 | **LLaMa-3.1-8B（本地）** | 零 API 成本，小任务适配能力足够 |
| 缓存介质 | **Redis Hash + Sorted Set** | O(1) 读写，原生支持 LRU、TTL、持久化 |
| 图编排 | **LangGraph** | 原生支持条件边、断点续跑、流式输出 |
| 容量策略 | **LRU（capacity≈关键词空间大小）** | 论文：超过唯一关键词数量后收益递减 |

**后续可探索方向**：
- 并行化 template_gen（论文提及）：缓存写入与响应返回并发执行，进一步降低延迟
- 推测性下一步预测（Speculative next-query inference）：提前为最可能的下一任务预生成模板
- 模糊匹配作为可选配置：允许用户通过 `fuzzy_threshold` 参数在召回率与准确率间权衡
- 多智能体架构扩展：跨 Agent 共享缓存，解决多 Agent 系统的缓存一致性问题
